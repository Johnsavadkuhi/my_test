How JavaScript Works - Simple Steps

1 JavaScript is single-threaded - It can only do one thing at a time
2 Normal tasks go straight to the Call Stack and run immediately
3 Waiting tasks (setTimeout, API calls) go to Web APIs to wait
4 When waiting tasks finish, they move to the Callback Queue
5 The Event Loop constantly checks if the Call Stack is empty
6 When the Call Stack is empty, the Event Loop moves tasks from Queue to Stack
7 This is why setTimeout(..., 0) doesn't run immediately - it waits in the queue!
