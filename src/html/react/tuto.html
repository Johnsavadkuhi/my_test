آشنایی با دو مفهوم اساسی در ری‌اکت و تفاوت‌های آن‌ها با مثال‌های تعاملی و انیمیشن

📤 Props (ویژگی‌ها)

Props داده‌هایی هستند که از یک کامپوننت والد به کامپوننت فرزند منتقل می‌شوند. این داده‌ها فقط-خواندنی (Read-only) هستند و کامپوننت فرزند نمی‌تواند آن‌ها را تغییر دهد.

ویژگی‌های Props:

  • ➡️ از والد به فرزند منتقل می‌شوند (یک‌طرفه)
  • 🔒 فقط-خواندنی هستند (Immutable)
  • 🎯 برای پیکربندی کامپوننت استفاده می‌شوند
  • 🔗 امکان ارتباط بین کامپوننت‌ها را فراهم می‌کنند

📝 State (حالت)

State داده‌هایی هستند که داخل یک کامپوننت مدیریت می‌شوند و می‌توانند در طول زمان تغییر کنند. وقتی state تغییر می‌کند، کامپوننت دوباره رندر می‌شود.

ویژگی‌های State:

  • 🔄 قابل تغییر هستند (Mutable)
  • 🏠 محلی به کامپوننت هستند
  • ⚡ تغییر state باعث رندر مجدد می‌شود
  • 💾 وضعیت پویای کامپوننت را مدیریت می‌کنند
